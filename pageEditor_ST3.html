Feedback page

Technical Description

The feedback page for our website uses semantic HTML elements to organize the content clearly and improve its accessibility as well. the <header> element contains the logo of the website and the navigation bar, the <nav> tag is used to group the navigation links to improve structure. The <main> tag holds the primary content ensuring clear seperation. <section> used to group program cards. each program is placed in a section with classes as card and program-card. <button> element provides interactive actions for the "Join us" and "Learn More" parts. A <form> was used for user input collection <fieldset> and <legend> were used to group personal details and feedback details, <label> tags were used to link inputs for clear field descriptions and <input> , <textarea> , <select> are used to collect user data <spanг elements with the class error are used to display validation messages. CSS Used : display: flex - used in .programs class to create horizontal scrolling layout for program cards. overflow-x: auto smooth scrolling, margin and padding used to create spacing. To ensure a consistent color theme background: linear-gradient(#00a8e8, #003459) was used to match the theme. rounded corners (border-radius) was applied to cards, buttons and forms for detail. box-shadow to create depth. :hover effects were applied to provide interactivity, ::-webkit-scrollbar used to style the scrollbar. Use of javascript for form validation. ensures correct inputs for name, email and category, real time character counter and error message displays were done using javascript.

Accessibility

Use of semantic HTML, labels and inputs for form accessibility, fieldset and legend to group inputs, error messages for validation use of alt text for all images, following a consistent theme through out the whole pages, interactive feedback - form validation, real time character count, success message.

Team Page

Technical Description

<header> element is used to contain the logo and navigation bar <main> holds the main content of the page <section> is used to display each members with the class team-card. <img> tags are used for the image showcase. CSS: (display:flex) is used in .team-container to align team cards (position:relative and absolute) creates the overlay effect on each card. .team-overlay is hidden using transform: translateY(100%) and is revealed on hover. (:hover) and (:focus) were used to create smooth animations. Transistions for smooth overlay.

Accessibility

Use of semantic HTML elements. Each team card includes tabindex="0" allow user for keyboard navigation alt texts per team member image for identification. :focus styling highlights the selected card for user ease of access.

Content Page

Technical Description

The <header> contains the logo and navigation menu. <main> section holds all the core content, The content is devided using <section> elements as (#content, #content2, #content3, #content4) <h1>, <h2>, <h3> are used for clear information titles. <ul> are used for key point presentation. <a href="content"> allows user to jump between sections. @media ensures images resize properly on smaller screens due to large image usage.

Accessibility

Use of semantic HTML elements helps readers understand the layout. alt texts describe each image for user ease. Use of @media for device responsiveness.
