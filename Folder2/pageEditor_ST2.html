Kumaresan Piragatheesan, Group 03, Splash Screen & AIS

Splash Screen

Technical Description

The Splash screen was implemented using HTML5 semantic elements such as <header>, <main>, and <footer>. CSS was used for a fixed background image with overlay effects, responsive typography, and a loader bar animation using @keyframes. Key elements like headings, paragraphs, and lists were structured for clarity, and buttons with hover effects improve interactivity.

Accessibility

Alt text was added to images for screen readers, color contrast ensures readability, and semantic HTML improves navigation for assistive technologies.

AIS

Technical Description

The Gallery page was built with a grid layout using CSS Grid (display: grid; and grid-template-columns) to arrange images responsively. Overlay effects and hover states were added with pseudo-elements ::before. Cards were implemented to encapsulate images and descriptions, with clear borders, padding, and shadows for visual distinction.

Accessibility

Each image includes descriptive alt text, the cards maintain readable text over backgrounds with overlays, and link and button focus states ensure keyboard navigation is possible.

Content Page

Technical Description

The Content Page uses semantic sections (<section>) with headings (<h2> and <h3>) to organize SDG 14 information. Internal anchor links provide quick navigation within the page. CSS styling includes responsive images with borders and shadows, a fixed “Go to Top” button, and a consistent color scheme matching the website template. Lists and paragraphs are styled for readability and accessibility.

Accessibility

All images include descriptive alt text, headings follow a logical hierarchy for screen readers, and contrast ensures text is readable. Anchor links and semantic structure improve navigation.

Content Page

Challenges and Lessons Learned

Challenges included ensuring a consistent template across pages and implementing internal navigation for long content. Solutions involved following the shared CSS and HTML structure strictly and testing anchor links and buttons. Lessons learned include better planning for accessibility, consistent styling, and teamwork coordination.

Compliance

All pages follow Janet regulations by using semantic HTML, accessible images, and external CSS for styling. Pages validate successfully and maintain consistent navigation and link structures. See lecture notes for compliance details.
